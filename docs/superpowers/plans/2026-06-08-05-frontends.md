# Plan 5 — Frontends (Invitee + Owner Admin) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build feature 8 — the two server-rendered Qute frontends — plus the UI halves of features 12–16. A **public invitee UI** (no auth) to browse public meeting types, pick a slot, book (with Cloudflare Turnstile + honeypot abuse protection), get a confirmation showing the meeting **location** (Google Meet link, or phone/in-person/custom detail), and manage (reschedule/cancel) via an **unguessable `manageToken` link**. Approval-type meeting types show "Request"/"pending owner approval" wording instead of an instant confirmation. An **owner/admin UI** (HTTP Basic, single configured admin) to view bookings, manage meeting types (including SECRET ones, with the new min-notice/horizon/location/approval fields), edit availability, manage **date-specific overrides**, work an **approval queue** (approve/decline PENDING bookings), edit settings (including the **owner-notify opt-out** + reminder lead-time), and connect Google. This plan also delivers the public half of feature 9: the public landing lists `MeetingType.listPublic()` and MUST NOT show secret types, while the admin meeting-types page lists `MeetingType.listAll()` and DOES show them.

**Architecture:** Two JAX-RS resource classes in `com.calit.web` render Qute templates via `@CheckedTemplate` (compile-time-checked, typed). Server-side, form-encoded POSTs (`application/x-www-form-urlencoded`) call the existing services/Panache **directly** (not via the JSON REST layer) — `MeetingType`/`OwnerSettings`/`AvailabilityRule`/`DateOverride` (Plan 1/1b) and `BookingService` (Plan 3). The public landing uses `MeetingType.listPublic()` (active && !secret); the booking page resolves any slug (secret included) via `MeetingType.findBySlug`. The invitee manage/reschedule/cancel routes key off the unguessable **`Booking.manageToken`** (UUID), NOT the numeric id — there are NO numeric-id booking routes anywhere in this plan. The admin group lives under `/admin/*`, gated by HTTP Basic against a single configured `admin` user/role via `quarkus-elytron-security-properties-file`; all public paths stay open. New admin pages: `GET /admin/date-overrides` (feature 12), `GET /admin/pending` approval queue (feature 14), plus the meeting-type form's new fields (11/13/14a) and the settings notify-toggle + reminder lead (14/15). `GET /admin/google` simply links to `GET /api/google/connect` (Plan 2). Plain HTML + a small shared CSS string. Client JS: the shared vanilla viewer-local-timezone snippet on the **invitee** pages that reformats already-rendered absolute instants into the *viewer's* local timezone (Calendly-standard — see the "Time model / Display timezone" bullet in the overview), plus the third-party Cloudflare **Turnstile** widget script on the booking form when enabled; the owner/admin pages remain pure server-rendered HTML in `OwnerSettings.timezone` with no viewer-local conversion.

**Display-timezone (invitee-only, frontend-only):** Per the overview's Time model, every invitee-facing page that shows times (the slot picker `GET /book/{slug}`, the confirmation page, and the manage/reschedule page) renders each time as an absolute instant in a machine-readable `data-utc="…"` attribute (derived from the slot/booking `ZonedDateTime`/`Instant` via `.toInstant().toString()`, an unambiguous absolute point in time) with an owner-tz-formatted server fallback inside the element. A tiny inline vanilla JS snippet (`Layout.TZ_SCRIPT`) detects the viewer zone (`Intl.DateTimeFormat().resolvedOptions().timeZone`), populates a `<select id="tz-picker">`, and reformats every `[data-utc]` element via `new Date(el.dataset.utc).toLocaleString(...)` on load and on picker change, showing "Times shown in: <zone>". The booking/reschedule form still submits the chosen slot's absolute UTC instant unchanged (hidden/radio value = the instant), so the viewer's display zone affects only the LABEL, never which instant is booked. The backend already exposes absolute instants — no backend change. **Owner/admin pages are unchanged: they intentionally do NOT do viewer-local conversion.**

**Tech Stack:** Java 25, Quarkus 3.35.3, Qute (`quarkus-rest-qute`), RESTEasy Reactive, `quarkus-elytron-security-properties-file`, JUnit 5, RestAssured. Tests run against Quarkus **Dev Services** (Testcontainers Postgres) — **Docker must be running**. `BookingService` depends on `CalendarPort` (`com.calit.google`, Plan 2/3); tests use `@InjectMock CalendarPort` to report connected, no busy (`freeBusy(...)` → empty), and a known `CreatedEvent` (with the Meet link) from `createEvent(...)`.

```java
package com.calit.web;

/**
 * Shared inline CSS (all pages) plus the invitee-only timezone-picker bar and reformat
 * script. Owner/admin pages use only {@link #CSS}; the invitee pages additionally use
 * {@link #TZ_BAR}/{@link #TZ_SCRIPT} to relabel times into the viewer's local zone.
 */
public final class Layout {

    private Layout() {}

    public static final String CSS = """
            body{font-family:system-ui,sans-serif;max-width:760px;margin:2rem auto;padding:0 1rem;color:#222}
            a{color:#2563eb}
            .card{border:1px solid #ddd;border-radius:8px;padding:1rem;margin:.75rem 0}
            .badge{background:#fde68a;border-radius:4px;padding:.1rem .4rem;font-size:.75rem}
            .slot{display:inline-block;margin:.2rem;padding:.4rem .6rem;border:1px solid #2563eb;border-radius:6px}
            nav a{margin-right:1rem}
            label{display:block;margin:.5rem 0}
            input,select{padding:.3rem}
            .err{color:#b91c1c}
            button{padding:.4rem .8rem;cursor:pointer}
            .tz-bar{margin:.5rem 0;font-size:.9rem;color:#444}
            """;

    /**
     * Shared inline vanilla JS reused by every invitee-facing page that shows times
     * (the slot picker, the confirmation page, and the manage/reschedule page).
     *
     * <p>The server renders each time as an absolute instant in a {@code data-utc}
     * attribute (an ISO-8601 instant with offset or {@code Z}); this script reformats
     * every {@code [data-utc]} element into the VIEWER's local timezone (auto-detected,
     * overridable via {@code #tz-picker}). It NEVER changes any submitted value — the
     * booking form's hidden {@code startUtc} input keeps its absolute UTC instant, so the
     * display zone only changes the LABEL, never which instant is booked.</p>
     *
     * <p>The stable marker comment {@code CALIT_TZ_REFORMAT} lets @QuarkusTest assert the
     * script is present without executing it (RestAssured can't run JS).</p>
     */
    public static final String TZ_SCRIPT = """
            <script>
            /* CALIT_TZ_REFORMAT — viewer-local time reformatting (Calendly-standard) */
            (function () {
              var ZONES = [
                'America/Los_Angeles','America/Denver','America/Chicago','America/New_York',
                'America/Sao_Paulo','UTC','Europe/London','Europe/Amsterdam','Europe/Berlin',
                'Europe/Paris','Europe/Madrid','Europe/Athens','Africa/Johannesburg',
                'Asia/Dubai','Asia/Kolkata','Asia/Singapore','Asia/Tokyo',
                'Australia/Sydney','Pacific/Auckland'
              ];
              var detected = Intl.DateTimeFormat().resolvedOptions().timeZone || 'UTC';
              if (ZONES.indexOf(detected) < 0) { ZONES.unshift(detected); }

              var picker = document.getElementById('tz-picker');
              var label  = document.getElementById('tz-label');
              if (!picker) { return; }
              ZONES.forEach(function (z) {
                var o = document.createElement('option');
                o.value = z; o.textContent = z;
                if (z === detected) { o.selected = true; }
                picker.appendChild(o);
              });

              function render() {
                var tz = picker.value;
                if (label) { label.textContent = tz; }
                document.querySelectorAll('[data-utc]').forEach(function (el) {
                  var d = new Date(el.dataset.utc);
                  var opts = (el.dataset.timeOnly === '1')
                    ? { timeStyle: 'short', timeZone: tz }
                    : { dateStyle: 'full', timeStyle: 'short', timeZone: tz };
                  el.textContent = d.toLocaleString([], opts);
                });
              }
              picker.addEventListener('change', render);
              render();
            })();
            </script>
            """;

    /** Reusable timezone-picker bar (detected zone selected client-side by TZ_SCRIPT). */
    public static final String TZ_BAR = """
            <div class="tz-bar">
              Times shown in: <strong><span id="tz-label">your local time</span></strong>
              <label style="display:inline">Change:
                <select id="tz-picker"></select>
              </label>
            </div>
            """;
}
```

> `TZ_SCRIPT`/`TZ_BAR` are passed to the three invitee templates as the `tzScript`/`tzBar` data params (raw HTML, rendered with Qute's `{tzScript.raw}`/`{tzBar.raw}` so the markup is emitted verbatim). They are NOT included on any owner/admin template — admin pages intentionally render in `OwnerSettings.timezone` only (see Task 6+ note).

**Behavior:** Resolve the slug via `MeetingType.findBySlug` (secret types ARE reachable here by direct link — do not block). 404 if missing. Compute `BookingService.availableSlots(type, today, today+N)` (N = 14 days). Render slots grouped by date, each as a radio option carrying the slot's UTC instant. After the slot picker, render the **booking form**: the always-present built-ins **Full name** (required) + **Email** (required), THEN one input per `BookingField.formFor(type.id)` in `position` order. The single form posts name + email + chosen instant + every custom answer to `POST /book/{slug}`.

**Abuse protection (feature 16):** the form ALWAYS carries a hidden **honeypot** field `<input name="website" style="display:none">` — a real human leaves it blank; a non-empty value is rejected server-side (Plan 3). When `calit.turnstile.enabled` is true the form ALSO renders a Cloudflare **Turnstile** widget: the `<div class="cf-turnstile" data-sitekey="{turnstileSiteKey}"></div>` placeholder plus the `https://challenges.cloudflare.com/turnstile/v0/api.js` script. Turnstile injects a hidden `cf-turnstile-response` input that the browser submits with the form. When the flag is false the widget + script are skipped (no `cf-turnstile` markup at all). Both the honeypot value (`website`) and the Turnstile token (`cf-turnstile-response`) are forwarded to `BookingService.book(...)`, which verifies them server-side and enforces the per-email/day cap (Plan 3). The resource passes the config flag + site key into the template (`@ConfigProperty`).

**Location (feature 13):** the booking page shows the meeting type's location BEFORE booking when it is fixed and shareable — `PHONE`/`IN_PERSON`/`CUSTOM` show `type.locationDetail`; `GOOGLE_MEET` shows "Google Meet — link sent after booking" (the actual Meet URL only exists once the event is created). The `LocationType` enum constant drives which line renders.

**Approval-type UX (feature 14a):** when `type.requiresApproval` is true the submit button says **"Request"** (not "Confirm booking") and the page makes clear it is a request pending owner approval; the post-submit page (Task 4) shows "Request sent — pending owner approval", not a confirmation. Auto (non-approval) types keep the "Confirm booking" button and the confirmed-with-location page.

**Viewer-local time display (feature 8 UX, Calendly-standard):** each slot radio carries BOTH the absolute UTC instant as its `value` (what gets submitted) AND a `data-utc="…"` attribute on a `<time>` element wrapping the visible label. `SlotView` exposes `startUtc` (the `Instant.toString()` value, used for the radio value, the `data-utc` attribute, AND chronological sorting) plus an owner-tz `label` rendered as the server fallback text inside the `<time>` element. The shared `Layout.TZ_BAR` ("Times shown in: …" + `#tz-picker`) and `Layout.TZ_SCRIPT` are rendered into the page so the slot times reformat to the viewer's detected zone on load and on picker change. Crucially, the picker only relabels the visible text — the radio `value` (the absolute UTC instant) is never touched, so the booked instant is timezone-independent. The same `startUtc` string is therefore the single source of truth for both the form value and the `data-utc` display attribute.

**Dynamic custom fields (feature 10):** resolve the EXTRA fields with `BookingField.formFor(type.id)` (already ordered by `position`). Map each `FieldType` to an input: `SHORT_TEXT`→`<input type="text">`, `LONG_TEXT`→`<textarea>`, `EMAIL`→`<input type="email">`, `PHONE`→`<input type="tel">`, `NUMBER`→`<input type="number">`; add the `required` attribute when `field.required`. Each custom input is named `answers.<fieldKey>` so the POST handler (Task 4) can reconstruct the `Map<String,String> answers` by stripping the `answers.` prefix. The built-in Full name/Email stay named `inviteeName`/`inviteeEmail` (mapped to `Booking.inviteeName`/`inviteeEmail`), NOT to `answers`.

    @Inject
    BookingService bookingService;

    // Owner-configurable Turnstile (feature 16). When disabled, the template skips the widget.
    @ConfigProperty(name = "calit.turnstile.enabled", defaultValue = "false")
    boolean turnstileEnabled;
    @ConfigProperty(name = "calit.turnstile.site-key", defaultValue = "")
    String turnstileSiteKey;

    private static final int BOOK_WINDOW_DAYS = 14;
    private static final DateTimeFormatter DATE_FMT = DateTimeFormatter.ofPattern("EEEE, d MMMM yyyy");
    private static final DateTimeFormatter TIME_FMT = DateTimeFormatter.ofPattern("HH:mm");

    /** One selectable slot: human label + the UTC instant string used as the form value. */
    public record SlotView(String label, String startUtc) {}

    @GET
    @Path("/book/{slug}")
    @Produces(MediaType.TEXT_HTML)
    public TemplateInstance book(@PathParam("slug") String slug) {
        MeetingType type = MeetingType.findBySlug(slug); // secret types reachable by direct link
        if (type == null) {
            throw new NotFoundException("No meeting type with slug " + slug);
        }
        ZoneId zone = ZoneId.of(OwnerSettings.get().timezone);
        LocalDate from = LocalDate.now(zone);
        LocalDate to = from.plusDays(BOOK_WINDOW_DAYS);

        // Group slots by date label, preserving chronological order.
        Map<String, java.util.List<SlotView>> byDate = new LinkedHashMap<>();
        for (TimeSlot slot : bookingService.availableSlots(type, from, to)) {
            String dateLabel = slot.start().format(DATE_FMT);
            byDate.computeIfAbsent(dateLabel, k -> new java.util.ArrayList<>())
                  .add(new SlotView(slot.start().format(TIME_FMT),
                                    slot.start().toInstant().toString()));
        }
        // Resolved EXTRA fields (per-type-else-global), already ordered by position.
        List<BookingField> fields = BookingField.formFor(type.id);
        // turnstileEnabled drives the widget; site key is public (rendered). The approval
        // flag (type.requiresApproval) + locationType/locationDetail are read off `type`
        // directly in the template for the button wording + location line.
        return Templates.book(type, byDate, fields, null, Layout.CSS,
                              Layout.TZ_BAR, Layout.TZ_SCRIPT,
                              turnstileEnabled, turnstileSiteKey);
    }

One instant, one source of truth — the viewer's display zone never affects which instant the radio submits. ```java
    @POST
    @Path("/book/{slug}")
    @Consumes(MediaType.APPLICATION_FORM_URLENCODED)
    @Produces(MediaType.TEXT_HTML)
    public TemplateInstance submitBooking(@PathParam("slug") String slug,
                                          @RestForm String startUtc,
                                          @RestForm String inviteeName,
                                          @RestForm String inviteeEmail,
                                          @RestForm String website,                 // honeypot
                                          @RestForm("cf-turnstile-response") String turnstileToken,
                                          MultivaluedMap<String, String> form) {
        MeetingType type = MeetingType.findBySlug(slug);
        if (type == null) {
            throw new NotFoundException("No meeting type with slug " + slug);
        }

        // Collect every "answers.<fieldKey>" form param into the answers map (strip the prefix).
        Map<String, String> answers = new HashMap<>();
        for (Map.Entry<String, java.util.List<String>> e : form.entrySet()) {
            if (e.getKey().startsWith("answers.")) {
                answers.put(e.getKey().substring("answers.".length()),
                            e.getValue().isEmpty() ? "" : e.getValue().get(0));
            }
        }

        Booking booking;
        try {
            // book(...) enforces required fields AND the abuse guards (Turnstile + honeypot +
            // per-email/day cap) server-side; the handler just forwards the two raw inputs.
            booking = bookingService.book(
                    slug, Instant.parse(startUtc), inviteeName, inviteeEmail, answers,
                    turnstileToken, website);
        } catch (BookingException be) {
            // Required-field 422 OR an abuse-guard rejection (filled honeypot / failed Turnstile /
            // per-email cap). Re-render the form inline with the message; do NOT 500, NOT confirm.
            ZoneId zone = ZoneId.of(OwnerSettings.get().timezone);
            LocalDate from = LocalDate.now(zone);
            LocalDate to = from.plusDays(BOOK_WINDOW_DAYS);
            Map<String, java.util.List<SlotView>> byDate = new LinkedHashMap<>();
            for (TimeSlot slot : bookingService.availableSlots(type, from, to)) {
                byDate.computeIfAbsent(slot.start().format(DATE_FMT), k -> new java.util.ArrayList<>())
                      .add(new SlotView(slot.start().format(TIME_FMT),
                                        slot.start().toInstant().toString()));
            }
            return Templates.book(type, byDate, BookingField.formFor(type.id),
                                  be.getMessage(), Layout.CSS,
                                  Layout.TZ_BAR, Layout.TZ_SCRIPT,
                                  turnstileEnabled, turnstileSiteKey);
        }

        // Server fallback label is owner-tz; the page also carries the booked instant as a
        // data-utc attribute so the shared script can relabel it to the viewer's zone.
        ZoneId zone = ZoneId.of(OwnerSettings.get().timezone);
        String when = booking.startUtc.atZone(zone)
                .format(DateTimeFormatter.ofPattern("EEEE, d MMMM yyyy 'at' HH:mm (z)"));
        String startUtcIso = booking.startUtc.toString(); // absolute instant for data-utc
        // Approval types come back PENDING → "request sent" page (no Meet link yet); auto types
        // come back CONFIRMED → location/Meet confirmation. The location line is built from the
        // type so PHONE/IN_PERSON/CUSTOM show locationDetail and GOOGLE_MEET shows the Meet link.
        boolean pending = booking.status == BookingStatus.PENDING;
        String location = (type.locationType == com.calit.domain.LocationType.GOOGLE_MEET)
                ? booking.meetLink : type.locationDetail;
        return Templates.confirmation(booking, type, pending, location, when, startUtcIso,
                                      Layout.CSS, Layout.TZ_BAR, Layout.TZ_SCRIPT);
    }
```

@Path("/admin")
@RolesAllowed("admin")
public class AdminResource {

    @CheckedTemplate
    public static class Templates {
        public static native TemplateInstance dashboard(List<Booking> upcoming, long pendingCount, String css);
    }

    @GET
    @Produces(MediaType.TEXT_HTML)
    public TemplateInstance dashboard() {
        // Upcoming confirmed bookings, soonest first. PENDING ones live in the approval queue
        // (Task 12, GET /admin/pending), not here.
        List<Booking> upcoming = Booking.list(
                "status = ?1 and startUtc >= ?2 order by startUtc",
                com.calit.booking.BookingStatus.CONFIRMED, java.time.Instant.now());
        long pendingCount = Booking.count("status = ?1", com.calit.booking.BookingStatus.PENDING);
        return Templates.dashboard(upcoming, pendingCount, Layout.CSS);
    }
}

> **Owner/admin pages intentionally do NOT do viewer-local timezone conversion.** Unlike the three invitee pages, the admin templates render times in the owner's own timezone (`OwnerSettings.timezone`) — it is the owner's schedule, not a visitor's.                 ? null : Long.valueOf(meetingTypeId); // empty = global

        // Zip parallel windowStart[]/windowEnd[] into windows; none → empty list = day off.
        java.util.List<String> starts = form.getOrDefault("windowStart", java.util.List.of());
        java.util.List<String> ends = form.getOrDefault("windowEnd", java.util.List.of());
        for (int i = 0; i < starts.size() && i < ends.size(); i++) {
            if (starts.get(i).isBlank() || ends.get(i).isBlank()) { continue; }
            DateOverrideWindow w = new DateOverrideWindow();
            w.dateOverride = o;
            w.startTime = java.time.LocalTime.parse(starts.get(i));
            w.endTime = java.time.LocalTime.parse(ends.get(i));
            o.windows.add(w);
        }
        o.persist(); // cascade persists the windows (Plan 1b mapping)
        return Templates.dateOverrides(
                DateOverride.<DateOverride>listAll(), MeetingType.listAll(), Layout.CSS);
    }

    @POST
    @Path("/date-overrides/{id}/delete")
    @Consumes(MediaType.APPLICATION_FORM_URLENCODED)
    @Produces(MediaType.TEXT_HTML)
    @Transactional
    public TemplateInstance deleteOverride(@PathParam("id") Long id) {
        DateOverride.deleteById(id);
        return Templates.dateOverrides(
                DateOverride.<DateOverride>listAll(), MeetingType.listAll(), Layout.CSS);
    }
```

The cascade from `DateOverride` to its `DateOverrideWindow` rows is defined in Plan 1b; if Plan 1b requires persisting windows explicitly, persist each `w` after `o.persist()` — the contract ("an override with these windows, or none = day off") is what matters.

## Built deviations (synced to reality)

- **Nested enums everywhere:** `LocationType` = `com.calit.domain.MeetingType.LocationType`, `FieldType` = `com.calit.domain.BookingField.FieldType` (the plan prose's top-level `com.calit.domain.*` forms don't exist).
- **`InjectMock`** = `io.quarkus.test.InjectMock`; **`maven.compiler.parameters=true`** added to pom (required by `@CheckedTemplate`/`@RestForm` name resolution).
- **Manage route** is `GET /booking/{manageToken}/manage` (+ `/reschedule`, `/cancel`), matching the Plan 4 email link; all invitee routes keyed by `manageToken` (no numeric-id invitee routes). The approval-queue approve/decline are the only numeric-id routes (owner action).
- **Date-relative test slots:** every test that exercises `availableSlots`/`book` uses `DAY = Instant.now().atZone(zone).toLocalDate().plusDays(7)` with the rule's day-of-week = `DAY.getDayOfWeek()` — fixed past dates would be dropped by the min-notice filter.
- **Booking exceptions:** no common superclass exists, so the booking POST multi-catches the four (`BookingValidationException`/`AbuseException`/`RateLimitException`/`BookingConflictException`) and re-renders the form inline.
- **Turnstile site-key** bound as `Optional<String>` (empty config value crashes a plain `@ConfigProperty String` under SmallRye).
- **Admin auth:** `src/test/resources/application.properties` pins the test admin password to `testpass` (role `admin`); admin pages carry both the path-based permission AND `@RolesAllowed("admin")`; each admin page has a real 401-unauthenticated test.
- **`slotIntervalMinutes` exposed** on the admin meeting-type form (the post-Plan-3 enhancement; blank → null).
- **`DateOverride.windows` is `@Transient`** (intentional, per the entity), so the admin create-handler persists each `DateOverrideWindow` explicitly with `dateOverrideId = override.id` (and deletes children before parent); a `getWindows()` accessor was added for Qute, and an `overridesWithWindows()` helper populates the transient for listing. Verified end-to-end: the write key matches `DateOverride.resolve()`'s read key, so admin-created overrides actually affect `SlotService` availability.
- **Test isolation:** the committing admin/invitee tests use unique slugs and `@AfterEach`/seed-time cleanup (notably deleting committed global `BookingField` rows so a leaked required field can't break later `book()` validation).
- **Known minor follow-up (non-blocking):** admin form handlers call `LocationType/FieldType/DayOfWeek.valueOf(...)` and `Integer/Long.valueOf(...)` on form input without try-catch, so a hand-crafted malformed value yields 500 not 400. Acceptable for the single trusted-owner admin surface (forms use selects/number inputs); revisit with bean-validation if hardening the admin API.
