# DaisyUI Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Pico CSS v2 with Tailwind CSS v4 + daisyUI 5 across all 16 HTML page templates, built with the Tailwind standalone CLI (no Node.js), keeping the server-rendered model and folding in the design-review fixes.

**Architecture:** A `src/main/css/input.css` entry file imports Tailwind, loads daisyUI as a node dependency (`@plugin "daisyui"`), defines two custom indigo themes (light/dark), and ports the irreducible custom widgets (calendar, slot pills, booking grid, admin shell) into an `@layer components` block. **Bun** compiles `input.css` → `src/main/resources/META-INF/resources/calit.css` (the same `/calit.css` URL served today) — locally via `bun run css:watch`, and in Docker via a dedicated `oven/bun` stage whose output is copied into the Java build. No JS ships at runtime; Maven does not build CSS. Templates are rewritten to daisyUI classes; dynamic Qute logic, JS hooks, and form contracts are preserved verbatim.

**Tech Stack:** Quarkus 3.36.1, Qute, Maven, Bun, Tailwind CSS v4 (`@tailwindcss/cli`), daisyUI 5, Liberica musl Docker images.

**Reference:** Spec at `docs/superpowers/specs/2026-06-11-daisyui-migration-design.md`. daisyUI component docs at `.claude/skills/daisyui/components/`.

**Pinned versions:** `tailwindcss` 4.x + `daisyui` 5.x, frozen by `bun.lock` (committed). `bun install` resolves the latest 4/5 at setup; the lockfile freezes them for reproducible builds.

**Preserve verbatim in every template rewrite:**
- All Qute expressions (`{x}`, `{#if}`, `{#for}`, `{display:of(...)}`, `{x.raw}`).
- Form `method`/`action`, and all `name=` attributes (`startUtc`, `inviteeName`, `inviteeEmail`, `answers.{key}`, `website` honeypot, `j_username`, `j_password`, etc.).
- JS-coupled DOM: `<div id="calendar" class="calendar">`, `.booking-grid`, `.day-slots[data-date]`, `.slot` + `<input type="radio" name="startUtc">` + `<time data-utc ...>`, and `[data-slug-name]`/`[data-slug-autofill]`. These classes are styled by `@layer components`, not replaced.
- The `{tzBar.raw}`, `{tzScript.raw}`, `{calScript.raw}` injections.
